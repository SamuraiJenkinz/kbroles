## Publishing and Approval Workflow

The approval workflow for a Technical Knowledge article is as follows:

1. Author completes all fields and content.
2. Author clicks the **Publish** button (upper-right corner of article).
3. ServiceNow notifies the Knowledge team that review is needed.
4. Knowledge team reviews — contacts author if changes are required before approving.
5. Any listed owner or Knowledge team manager approves via the Owners tab.
6. Article moves to **Published** state.
7. Valid to date auto-sets to 365 days from publication.

Key rules that apply throughout the workflow:

- **Cannot edit a Published KB directly** — must check out first (creates a draft; live version stays visible until the new version is approved).
- **Permalinks only** — when linking to a KB article, always use Copy Permalink at the bottom of the article. It always references the latest version regardless of revision history.

## Edit / Retire / Delete Lifecycle

**Editing an article:**

1. Open the article and click **Edit** (upper-right).
2. Click **Checkout** — this creates an editable draft. The live version remains visible until the new version is approved.
3. Make changes, then click **Save** and **Publish**.
4. Knowledge team reviews and approves.
5. New version goes live; previous version is visible until the approval cycle completes.

SME restriction: only the listed SME (or the author if no SME is listed) may submit edits. Other users wishing to request changes should flag the article (see Flagging Articles).

**Retiring an article:**

1. Check out the article (same as editing).
2. Select the **Retire** option (upper-right).
3. Retirement follows the same approval process as publishing.
4. Once retired, the article is removed from publication and not searchable without filtering for unpublished articles.
5. Hyperlinks to retired articles still work — manually remove references in any other articles that linked to the retired one.

**Deleting an article:**

- Deletion is only available on draft or retired articles.
- A Published article must be checked out and retired before the delete option appears.
- An article currently in approval must be recalled before delete is available.
- **Deleted KBs cannot be recovered** — only delete when content has absolutely no further use.

## Flagging Articles

Any user with read access can flag an article. Use the flagging workflow when content is incorrect or outdated and you are not the designated SME.

Steps:

1. Click the **Flag Article** button in the article header.
2. A new window opens — enter the reason (as detailed as possible).
3. Click Submit.
4. ServiceNow creates a **knowledge feedback task** record.
5. The CTSS Knowledge team reviews and actions the task.

Flagging is the correct path for any content concern raised by a non-SME. Editing an article directly is reserved for the listed SME or the Knowledge team.

## Knowledge Blocks (Knowledge Team Only)

Knowledge Blocks are reusable HTML content snippets that can be inserted into multiple articles. Updating the block updates it everywhere it is inserted. Authoring access is restricted to the Knowledge team.

**To create a Knowledge Block:** Navigate to Knowledge > Knowledge Blocks in the ServiceNow application navigator.

**To insert a Knowledge Block into an article:** In the article, place the cursor where the block should appear → click the **Add Blocks** button → select the block → Insert to article.

**Lifecycle:** Knowledge Blocks follow the same draft/approve/publish/retire rules as standard articles.

Example use case: a Password Policy block inserted into any article that references password management ensures all such articles reflect the latest policy when the block is updated.

## Colleague Knowledge Criteria Check

Before approving any technical article, the Knowledge team checks the following criteria:

1. Do relevant Colleague-facing articles exist?
2. If yes — do they align with the latest technical information?
3. If not aligned — email `technology@mmc.com` noting observations, and add to approval comments.
4. If no colleague-facing articles exist and the topic warrants it — email `technology@mmc.com` and note in approval comments.

When emailing the comms team, include: support documentation for the application/service/product, self-service resolution steps, and the SME to associate with the article.

This criteria check is part of the approval responsibility and is performed by the Knowledge team, not the author.
