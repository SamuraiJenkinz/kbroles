## Decisions Made

- **Preserve wire error codes verbatim.** Renaming the HTTP body.error string (e.g. `token_expired` → `session_expired`) would have propagated a ~30-assertion ripple across ErrorCard + useChatStream + ChatSurface tests + the existing tests-e2e specs. The wire contract is the frontend's stable API; the internal discriminant is a backend-only naming refresh.
- **Session-present-but-empty vs no-cookie-at-all split.** The middleware returns `session_expired` only when the kb_session cookie is on the request but iron-session produces an empty session object. This allows Plan 05's frontend to differentiate "your session timed out — sign back in" from "you're not signed in — sign in" without a new wire code.
- **Dev-stub carries full role set.** Returning `roles: ['KbAssistant.User']` in the dev stub means role-gated code (future role checks in Plan 05+) lights up automatically in local dev. Without it, every locally-served request would 403 and defeat the purpose of the stub.
- **Chat route test mock rewritten hermetically.** The old `vi.importActual` delegation loaded the real middleware, which now calls `next/headers.cookies()` — throws outside a real request context. Options: (a) polyfill next/headers in this test file, (b) re-implement branches hermetically. Chose (b) because the real middleware's behaviour is thoroughly covered in its OWN test file; this test should be about the ROUTE translating AuthResults, not about the middleware's cookie semantics.
